Agent Harness for Computer Control

用于控制计算机的主流 agent harness——GUI 视觉操控、浏览器自动化、终端/代码执行三大家族的架构深度对比

2026-07-13 7 组核心论断 · high confidence 一手厂商资料 + arXiv 基准

00核心洞察

所有 computer-control agent 本质是同一件事：一个被增强的 LLM（工具 + 检索 + 记忆）跑「感知—推理—动作」反馈循环，每一步都从环境获取 ground truth 来自我纠错（Anthropic《Building Effective Agents》）。三大家族的真正分野只在感知层与 grounding（定位）机制——这决定了其余一切。

一句话总览：三者不是竞品，而是感知层光谱——能拿到文本就别碰 DOM，能拿到 DOM 就别去数像素。视觉操控是「最后手段的通用层」，代价是承受 grounding 的不确定性。实践趋势是分层混合：终端/浏览器 harness 处理有结构的部分，仅在无结构的封闭 GUI 处回退到视觉操控。

01三大架构家族

家族 ①

GUI 视觉操控

  • Anthropic Computer Use
  • OpenAI CUA / Operator
  • UI-TARS
  • 感知：纯截图（周期帧）
  • 动作：虚拟鼠标+键盘
家族 ②

浏览器自动化

  • browser-use
  • Skyvern
  • Playwright MCP
  • 感知：DOM / CDP 结构
  • 动作：高层交互（click/fill）
家族 ③

终端 / 代码 harness

  • Claude Code
  • OpenAI Codex CLI
  • OpenHands
  • 感知：纯文本 / 执行结果
  • 动作：shell + 文件编辑 + 工具

02跨维度对比矩阵

六大架构维度横向对比，差异根源都可追溯到感知层的选择。

维度 ① GUI 视觉操控 ② 浏览器自动化 ③ 终端 / 代码
感知层 纯截图，周期性帧（非视频流） 结构化页面数据（DOM/CDP），截图可选 纯文本：终端输出、文件、代码执行结果
grounding 机制 像素 → 坐标映射（核心瓶颈） DOM 元素索引，无需空间坐标 无需空间定位（路径/行号即地址）
动作空间 通用虚拟鼠标+键盘（像素级 click/type/scroll） 高层交互（open/click/type/fill，作用于索引元素） shell 命令 + 文件编辑 + 工具调用
Agent Loop 截图入上下文 → CoT → 动作 → 重观察 → 纠错重试 NL 任务 → 自主多步循环 → 返回动作历史 写代码/命令 → 执行 → 读结果 → 迭代
通用性 vs 精度 通用性最强（任何 GUI），精度最弱 仅限浏览器，但页面内精度高、稳 仅限可脚本化环境，精度最高
安全 / 沙箱 虚拟机 / 远程桌面隔离 独立浏览器上下文 Docker 沙箱运行时（原生一等设计）

03① GUI 视觉操控 — grounding 是决定性难题

三家独立厂商架构惊人趋同：只看截图、通过虚拟鼠标键盘行动。真正差异在如何把感知到的 UI 映射到精确动作坐标。

Anthropic
训练模型数像素（counts pixels）来移动光标。官方明说「训练 Claude 准确数像素是关键」——直接的坐标回归路线。
OpenAI CUA
监督学习负责感知与输入控制，强化学习负责推理、纠错与恢复。RL 专门赋予「UI 状态异常时的恢复行为」。
UI-TARS
原生端到端模型，在大规模动作轨迹上训练，把动作标准化为跨平台统一空间——感知-推理-动作全在一个模型内，不做外部编排。
已知硬伤：截图是离散帧，会漏掉两帧之间转瞬即逝的动作/通知。OSWorld 基准（arXiv 2404.07972）证实 agent「主要卡在 GUI grounding」，坐实这是该家族的核心瓶颈。

04② 浏览器自动化 — 绕开像素，赌结构化数据

两个截然不同的设计点，对应快而脆 vs 抗变但贵的权衡。

browser-use

DOM / CDP 路线

  • 走原始 CDP（Chrome DevTools Protocol）
  • 对索引后的可交互元素下手，非像素点击
  • 截图仅作可选视觉上下文
  • Agent(task=…, llm=…) + await agent.run()
  • 优点：快、不依赖坐标；代价：绑定 DOM 结构
Skyvern

视觉优先路线

  • 以 Vision LLM 为主感知交互
  • 「不用预设 XPath / selector」
  • 用一群协作 agent（swarm）理解网页、规划执行
  • 目标：躲开脆弱的 XPath——页面改版即失效
  • 代价：更贵、更慢，但抗页面变化

05③ 终端 / 代码 harness — 最成熟、最可靠

「LLM 用工具跑反馈循环」最纯粹的形态，没有截图也没有 DOM 感知层。

OpenHands
agent 通过写代码、操作命令行、浏览网页行动，配 Docker 沙箱运行时做隔离——安全性是一等公民（arXiv 2407.16741）。
Codex CLI
本地终端里跑的轻量编码 agent，codex 命令启动，纯 shell / 工具调用 harness。
为何最可靠：文本地址（文件路径、行号、命令）天然精确，根本不需要空间 grounding，从源头绕开了家族①的瓶颈。ground truth 来自代码执行结果，反馈信号强而干净。

06选型结论

维度最优家族说明
通用性（任意软件）① 视觉唯一能操作无 API / 无 DOM 的封闭 GUI
可靠性 / 精度③ 终端文本地址精确，无 grounding 误差
Web 任务性价比② 浏览器有 DOM 时远比截像素高效
安全隔离成熟度③ 终端沙箱 / Docker 是原生设计（OpenHands）
核心技术瓶颈① 视觉grounding 被 OSWorld 证实为主要失败源
